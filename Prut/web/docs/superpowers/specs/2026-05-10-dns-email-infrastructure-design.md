# DNS & Email Infrastructure for peroot.space

**Date:** 2026-05-10
**Author:** Gal Sasson + Claude
**Status:** Approved, ready for implementation

---

## Problem

`peroot.space` has no SPF, no DMARC, and no DKIM. Anyone can spoof `@peroot.space` in From headers. Transactional email currently goes out from `no-reply@joya-tech.net` via Resend, creating a brand mismatch with the user-facing `peroot.space` domain. There is no working mailbox at `peroot.space` for human replies.

## Goals

1. Working inbox at `info@peroot.space` accessible from existing Gmail
2. Transactional sends from `noreply@peroot.space` via Resend, fully authenticated
3. SPF + DKIM + DMARC pass on every outbound email from the domain
4. Single brand identity (peroot.space) on all outbound mail
5. Zero additional monthly cost

## Non-Goals

- Migrating DNS provider (stays on Cloudflare; consolidation at Vercel rejected as no real benefit)
- Migrating registrar (stays on Namecheap)
- Replacing Resend with another transactional provider
- Changing the underlying Google Workspace account (joya-tech.net remains the parent)

## Architecture

| Layer | Provider | Notes |
|---|---|---|
| Registrar | Namecheap | nameservers point to Cloudflare |
| Authoritative DNS | Cloudflare | zone id `53867c3350b519d3b1fe6653b75307cb` |
| Mailbox (receive) | Google Workspace Business Standard | `peroot.space` added as **secondary domain** to existing joya-tech.net account; `info@peroot.space` becomes alias of `gal@joya-tech.net` |
| Transactional sending | Resend (Free tier) | `joya-tech.net` removed; `peroot.space` added |
| Application config | Vercel env vars | `RESEND_FROM_EMAIL`, `RESEND_REPLY_TO` updated |

## Final Email Identities

| Address | Purpose | Channel |
|---|---|---|
| `info@peroot.space` | Human contact, replies to transactional | Gmail (alias of gal@joya-tech.net) |
| `noreply@peroot.space` | Application transactional (welcome, password reset, payment, daily digest) | Resend |
| `gal@peroot.space` (optional) | Personal | Gmail alias |

## DMARC Rollout

Three-stage progression to avoid breaking legitimate mail:

1. **Now → +14d:** `p=none` (monitor only). Reports go to gal@joya-tech.net daily.
2. **+14d → +45d:** `p=quarantine` once two weeks of reports show all legitimate sends pass SPF + DKIM.
3. **+45d onward:** `p=reject` for full anti-spoofing protection.

Re-evaluate at each stage; if any report shows a legitimate-but-failing source, fix authentication before advancing.

## Code Changes

### `src/lib/emails/service.ts:15-16`

```diff
- private static from = process.env.RESEND_FROM_EMAIL || 'no-reply@joya-tech.net';
- private static defaultReplyTo = process.env.RESEND_REPLY_TO || 'gal@joya-tech.net';
+ private static from = process.env.RESEND_FROM_EMAIL || 'noreply@peroot.space';
+ private static defaultReplyTo = process.env.RESEND_REPLY_TO || 'info@peroot.space';
```

The fallback shifts to peroot.space so a missing env var fails closed (consistent with new sender) rather than at the old domain that no longer exists in Resend.

### Vercel env vars (Production + Preview)

```
RESEND_FROM_EMAIL=noreply@peroot.space
RESEND_REPLY_TO=info@peroot.space
```

### Out of scope for this change

User-facing contact email references (`gal@joya-tech.net` in footer, FAQ, privacy, contact page, schema.org, llms.txt) are NOT changed. They reference the human inbox, which continues to exist on Google Workspace under joya-tech.net. Changing them is a separate brand-decision task.

## Execution Order

The order matters because Resend Free tier allows only one domain.

1. **[Cloudflare]** Add SPF (`v=spf1 ... ~all`) — safe; no effect on current sends.
2. **[Cloudflare]** Add DMARC `p=none` — safe; monitoring only.
3. **[User in Google Admin]** Add peroot.space as secondary domain; receive verification TXT.
4. **[Cloudflare]** Add Google site-verification TXT for the secondary domain.
5. **[User in Google Admin]** Verify; enable Gmail; add info@ alias to gal@joya-tech.net; generate DKIM.
6. **[Cloudflare]** Add `google._domainkey` TXT.
7. **[Resend]** Delete joya-tech.net; add peroot.space; receive DKIM + Return-Path records.
8. **[Cloudflare]** Add Resend DKIM + send subdomain records.
9. **[Resend]** Verify domain (auto after DNS propagates, typically <15 min).
10. **[Code]** Update fallbacks in `src/lib/emails/service.ts`.
11. **[Vercel]** Set `RESEND_FROM_EMAIL` and `RESEND_REPLY_TO`; trigger production deploy.
12. **[User in Gmail]** Settings → Accounts → "Send mail as" → add `info@peroot.space`; confirm.
13. **[Verification]** mxtoolbox SPF/DMARC/DKIM checks; mail-tester.com end-to-end score ≥9/10.
14. **[+14 days]** Promote DMARC to `p=quarantine`.
15. **[+45 days]** Promote DMARC to `p=reject`.

**Breakage window:** Steps 7–11 cause transactional sends to fail until verification completes and the env is updated (~5–20 minutes). Schedule during low-traffic.

## Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Resend domain verification fails after add | Cloudflare propagates fast (<2 min usually); retry; manually re-trigger via dashboard |
| Code deploy lands before Resend verifies → all sends fail | Sequence step 11 AFTER step 9 confirms verified |
| DMARC `p=reject` shipped too aggressively → legitimate mail bounces | Strict three-stage rollout; check reports at each stage |
| User forgets to do Gmail "Send mail as" → can reply but Reply-To shows joya-tech | Step 12 explicitly required; verification email from Google must be confirmed |
| SPF too long (>255 chars) | Current SPF is short; no issue. Stay below 10 DNS lookups by avoiding nested includes |
| Old Resend domain joya-tech.net still in use by other tenants/projects | Confirmed not used by any joya-tech project (this Resend account is dedicated to Peroot) |

## Success Criteria

- `dig TXT peroot.space` returns SPF with both Google + Resend includes
- `dig TXT _dmarc.peroot.space` returns DMARC record
- `dig TXT google._domainkey.peroot.space` returns Google DKIM key
- `dig TXT resend._domainkey.peroot.space` returns Resend DKIM key
- mail-tester.com score ≥ 9/10 for both `info@peroot.space` and `noreply@peroot.space`
- DMARC daily aggregate reports arrive at gal@joya-tech.net
- A test transactional email from production (e.g. password reset) arrives with `From: noreply@peroot.space`, all auth headers pass
