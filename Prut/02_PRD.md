# [מסמך 2] PRD — Product Requirements Document

### 1. Scope Definition
| Feature | MVP | V1 | V2 |
| :--- | :--- | :--- | :--- |
| **Prompt Improver** | בסיסי (קלט->פלט) | כולל שאלות הבהרה | מצב Interactive Chat |
| **Language Support** | עברית, אנגלית | דו-לשוני משולב | שפות נוספות |
| **Templates** | 8 קטגוריות קבועות | חיפוש + יצירה אישית | Marketplace |
| **Auth** | Google Login / Email | ארנק קרדיטים | SSO ארגוני |
| **History** | מקומי (Local Storage) | שמירה בענן (DB) | תיקיות וניהול |
| **Export** | העתקה (Copy) | שיתוף לינק | ייצוא ל-PDF/Notion |

### 2. User Personas
1.  **דנה המשווקת (30):** צריכה 10 וריאציות לפוסט ב-5 דקות. אין לה כוח לריב עם ChatGPT. היא רוצה "קסם".
2.  **רון המפתח (26):** צריך פרומפט ל-Code Review אבל מסתבך עם הניסוח באנגלית טכנית. רוצה מבנה לוגי.
3.  **גלית מנהלת משאבי אנוש (45):** צריכה לנסח מכתב רגיש לעובד. רוצה טון "רשמי אבל אמפתי".

### 3. User Journeys (Core Loop)
**Journey A: השיפור המהיר**
1. משתמש נכנס ל-Homepage.
2. מקליד בתיבה: "תכתוב מייל מכירות ללקוח שלא ענה".
3. לוחץ "פרט לי" (Enhance).
4. מקבל פרומפט משופר + הסבר קצר על השיפור ("הוספתי פרסונה של איש מכירות והגדרתי טון אסרטיבי-רך").
5. מעתיק את הפרומפט.

### 4. Feature List (MVP Priorities)
*   **Must:** מנוע שיפור פרומפטים (Backend Logic), ממשק צ'אט/תיבה, העתקה לקליפבורד, תמיכה מלאה ב-RTL, לוגין בסיסי (לשמירת היסטוריה), 8 קטגוריות תבנית.
*   **Should:** שמירת פרומפטים למועדפים, Dark Mode, בחירת "רמת יצירתיות".
*   **Could:** שיתוף פרומפט ברשתות חברתיות, וריאציות אוטומטיות (3 גרסאות לבחירה).

### 5. תמחור ראשוני (Pricing)
*   **המלצה:** Freemium.
*   **Free:** עד 10 שיפורים ביום, גישה לכל התבניות הבסיסיות.
*   **Pro (49 ₪/חודש):** ללא הגבלה, גישה למודל חכם יותר (למשל GPT-4o במקום GPT-4o-mini), היסטוריה לא מוגבלת, תמיכה בוריאציות.

### 6. Non-Functional Requirements
*   **Latency:** תשובה ראשונית תוך פחות מ-3 שניות (Streaming).
*   **RTL:** תמיכה מושלמת (כולל סימני פיסוק מעורבים עברית/אנגלית).
*   **Mobile First:** האתר חייב להיות שמיש לחלוטין בנייד.
