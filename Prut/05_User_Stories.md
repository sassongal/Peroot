# [מסמך 5] User Stories + Acceptance Criteria

### דוגמאות נבחרות (מתוך 20)

**Story 1: שיפור פרומפט בסיסי**
*   **כמשתמש** אני רוצה להזין טקסט גולמי ולקבל גרסה מפורטת, **כדי** שאקבל תוצאות טובות יותר מה-LLM שלי.
*   **AC:**
    *   Given: המשתמש הזין "כתוב מייל ללקוח" ולחץ "שפר".
    *   When: המערכת מעבדת את הבקשה.
    *   Then: מוצג פרומפט של לפחות 3 פסקאות הכולל: פרסונה, הקשר, משימה, ופורמט פלט.
    *   Then: מוצג הסבר קצר על מה שופר.

**Story 2: העתקה מהירה**
*   **כמשתמש** אני רוצה להעתיק את התוצאה בקליק אחד, **כדי** להדביק אותה מיד בכלי ה-AI שלי.
*   **AC:**
    *   Given: קיים פרומפט בתוצאה.
    *   When: לחיצה על אייקון ההעתקה.
    *   Then: הטקסט מועתק ללוח ומופיע חיווי ויזואלי (Toast/Icon change) למשך 2 שניות.

**Story 3: בחירת קטגוריה**
*   **כמשתמש** אני רוצה לבחור שהפרומפט הוא מסוג "שיווק", **כדי** שהשיפור יכלול טרמינולוגיה שיווקית מתאימה.
*   **AC:**
    *   Given: המשתמש בחר "שיווק" ב-Dropdown.
    *   When: לוחץ "שפר".
    *   Then: הפרומפט שנוצר כולל הנחיות כמו "השתמש במודל AIDA" או "התמקד בהנעה לפעולה".
