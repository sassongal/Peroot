# [מסמך 11] Security + Privacy

### Threat Model
*   **Abuse:** משתמש שמריץ סקריפט כדי לגמור לנו את מכסת ה-Tokens. -> פתרון: Rate Limiting חזק + Captcha ברישום.
*   **Prompt Injection:** משתמש שמנסה לגרום ל-Peroot להדליף את ה-System Prompt שלו. -> פתרון: הוראות חזקות ב-System Prompt לא לחשוף את ההוראות הפנימיות.

### פרטיות
*   לא שומרים פרטי כרטיס אשראי (הכל ב-Stripe).
*   תנאי השימוש יבהירו: "הפרומפטים שהזנתם עשויים לעבור עיבוד דרך צד שלישי (OpenAI), אל תזינו מידע סודי/פרטי".
